# Guia de Contribuição para o ScreenFlow PRO

Bem-vindo ao ScreenFlow PRO! Agradecemos o seu interesse em contribuir para o nosso projeto de código aberto. Suas contribuições são valiosas para o aprimoramento e crescimento do aplicativo.

## Como Contribuir

Existem diversas maneiras de contribuir para o ScreenFlow PRO:

*   **Reportar Bugs:** Se você encontrar algum bug ou problema, por favor, abra uma issue em nosso repositório GitHub, descrevendo detalhadamente o problema e os passos para reproduzi-lo.
*   **Sugerir Recursos:** Tem uma ideia para um novo recurso ou melhoria? Abra uma issue para discutir sua sugestão.
*   **Enviar Pull Requests:** Se você deseja contribuir com código, siga as diretrizes abaixo para enviar um Pull Request (PR).

## Diretrizes para Pull Requests

Para garantir um processo de revisão eficiente e consistente, por favor, siga estas diretrizes ao enviar um Pull Request:

1.  **Fork o Repositório:** Faça um fork do repositório `MANOSHOXFF/ScreenFlow-PRO-Official` (ou o nome final que for definido) para a sua conta GitHub.
2.  **Crie uma Branch:** Crie uma nova branch a partir da branch `main` para o seu trabalho. Use um nome descritivo para a branch (ex: `feature/nova-funcionalidade` ou `bugfix/correcao-erro-x`).
3.  **Faça Suas Alterações:** Implemente suas alterações ou novos recursos. Certifique-se de que seu código esteja limpo, bem comentado e siga as convenções de codificação existentes.
4.  **Testes:** Se aplicável, adicione testes para suas alterações. Certifique-se de que todos os testes existentes continuem passando.
5.  **Commits Significativos:** Faça commits pequenos e atômicos, com mensagens claras e descritivas que expliquem o propósito de cada commit.
6.  **Atualize a Documentação:** Se suas alterações afetarem a funcionalidade ou o uso do aplicativo, atualize a documentação relevante (README.md, etc.).
7.  **Sincronize com a `main`:** Antes de enviar seu PR, certifique-se de que sua branch esteja sincronizada com a branch `main` do repositório original para evitar conflitos.
8.  **Abra um Pull Request:** Abra um Pull Request para a branch `main` do repositório original. Inclua uma descrição clara e concisa das suas alterações, o problema que ele resolve ou o recurso que ele adiciona.
9.  **Revisão:** Esteja preparado para responder a perguntas e fazer ajustes com base no feedback da revisão do código.

## Código de Conduta

Esperamos que todos os contribuidores sigam nosso Código de Conduta para garantir um ambiente acolhedor e respeitoso para todos.
